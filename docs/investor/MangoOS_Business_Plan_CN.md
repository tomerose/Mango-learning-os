## 1. 执行摘要

**一句话定位：** MangoOS 是面向学生和学习型用户的 AI 学习与任务成果操作系统 — 将课程、资料、问题和目标转化为可交付、可复用、可导出的结构化成果。

**核心用户：** 大学生、备考者、留学生、内容生产者

**解决的问题：**
- AI 工具泛滥，但学生无法将它们转化为真正的学习成果
- 学习过程缺少闭环：输入→处理→输出→沉淀→复用
- 期末复习、论文写作、资料整理等高频场景缺少专用 AI 工具
- 现有 AI 聊天产品只能对话，不能生成可交付的 PDF、讲义、学习包

**当前阶段：** 早期实验产品，MVP 功能验证阶段。30+ 页面、24 组 API 路由、5 张数据库核心表已部署运行。校园场景尚未开始规模化验证。

**未来目标：** 成为学生从"使用 AI"到"用 AI 产出成果"的默认入口，覆盖 Web → 小程序 → App 多端。

## 2. 项目背景与问题

### 2.1 市场现状：AI 工具多，学生获得感低

2024-2026 年，以 ChatGPT、DeepSeek 为代表的对话式 AI 快速普及。学生在日常中频繁使用 AI 聊天，但痛点集中：

**痛点 1：对话不产出成果。** 学生用 AI 聊天提问后，得到的是一段文字回复 — 没有结构、没有排版、无法保存复用、无法导出为可提交的文档。聊完即忘，反复提问。

**痛点 2：学习过程碎片化。** 学生的 AI 使用分散在多个工具中：用 ChatGPT 问问题、用 Notion 记笔记、用 Quizlet 做闪卡、用 WPS 写文档。工具之间靠手动搬运，没有串联成闭环。

**痛点 3：期末复习场景未被 AI 有效服务。** 期末备考是学生最高频、最焦虑的刚需场景。现有工具要么是通用 AI 聊天（不会针对课程），要么是传统笔记/题库（没有 AI 生成能力）。缺少一个能根据课程信息自动检索资料、生成结构化复习讲义的专用工具。

**痛点 4：AI 输出质量不可控。** 学生直接用 AI 生成的内容往往存在逻辑混乱、缺乏引用来源、格式粗糙、不可交付等问题。需要一个中间层做质量审核。

**痛点 5：校园用户门槛敏感。** 学生用户对价格高度敏感，需要免费可用、付费增值的模式。注册流程不能复杂。

### 2.2 为什么是现在

- 开源大模型（DeepSeek 等）的推理成本已降至学生可接受范围
- Vercel、Supabase 等 Serverless 基础设施使个人开发者可独立部署生产级应用
- 中国高校学生群体对 AI 学习工具的接受度在 2024-2025 年出现跃升
- "AI 原生学习方式"从概念进入实用阶段，但市场尚无明确领导者

## 3. 产品定位

### 3.1 定位陈述

> **MangoOS = 学习任务输入 → 多源资料检索 → 结构化内容生成 → 质量审核 → PDF/文档导出 → 历史成果沉淀 → 复用的完整操作系统。**

MangoOS 不是聊天机器人，不是笔记工具，不是题库平台。它是一个**以学习成果生产为核心的操作系统**。

### 3.2 架构哲学

- **输入层**：文本、文件上传（PDF/Word/MD）、URL 抓取、关键词搜索
- **检索层**：多源 Research Orchestrator（11 个数据源：DuckDuckGo、Tavily、Jina Reader、GitHub、arXiv、Bilibili、Open Library 等）
- **生成层**：DeepSeek API（可插拔，支持 OpenAI 兼容的任何模型）
- **质量层**：Quality Gate v4 — 7 个验证维度 + 90 分硬门禁（Pro 用户）
- **输出层**：结构化文档、18 章节复习讲义、闪卡、练习题、学习计划
- **导出层**：HTML / Markdown / Word / PDF（ezPDF 服务器端 PDF）
- **存储层**：Supabase PostgreSQL + RLS（云端）或 localStorage（游客模式）
- **呈现层**：响应式 Web（桌面/平板/手机），暗色/浅色双模式，PWA 可安装

| 输入→成果闭环 | ❌ | 部分 | 部分 | ✅ |
| 多源资料检索 | ❌ | ❌ | ❌ | ✅ (11 源) |

## 4. 核心功能

### 4.1 MangoAgent — 任务成果生成引擎

MangoOS 的核心差异化模块。用户的自然语言任务输入 → Agent 自动判断是否需要联网检索 → 多源并行搜索 → AI 综合生成 → Quality Gate 审核 → 不合格自动加深（最多 2 轮）→ 结构化文档输出。

**关键指标（V14.8.1）：**
- Pro/Admin 用户强制执行资料检索（不可跳过）
- 质量评分 < 90 分 → 状态标记为 FAILED，显示未达标维度明细
- 输出自动保存到 Library（outcome_documents 表）
- 来源数量：Standard 4 条 / Pro 12 条
- 搜索方向：Standard 4 个 / Pro 8 个

### 4.2 Final Exam Master Review — 期末复习讲义

学生输入课程名称 + 学校 + 考试范围 + 上传课件 → 系统自动检索多源资料 → AI 生成 18 章节完整复习讲义：

封面、目录、课程概述、考试范围地图、知识图谱、分章节精讲、逻辑框架、高频考点、公式表、解题方法、典型例题、易错陷阱、检查清单、复习计划、模拟试卷、答案解析、冲刺手册、参考文献。

**当前状态：** API 功能完整，UI 已连接，支持 Word/PDF/MD/HTML 导出。导出质量已验证。

### 4.3 Mind Garden — 心理树洞与成长记录

10 种模式（日记、倾诉、结构化反思、CBT 认知重构、接地练习、呼吸训练、睡眠指导、自我关怀、压力恢复、情绪报告），双层安全防护（客户端 + 服务端危机检测），隐私优先（默认本地模式，明文知情同意才上云）。

**Codex P0 安全审计已通过。**

### 4.4 内容结构化引擎

Quality Gate v4 — 7 个验证维度：
1. 内容相关性（是否与主题匹配）
2. 来源可信度（是否有真实引用）
3. 结构完整性（是否有逻辑框架）
4. 内容充实度（是否有深度而非泛泛而谈）
5. 反通用化（是否避免了 AI 套话）
6. 格式化质量（是否有排版层级）
7. 可操作性（是否有行动建议）

新增维度（v4）：requiredFixes（需修复项）、needsAdminReview（需人工审核）、citationCount（引用数量）。

## 5. 核心竞争优势

### 5.1 场景优势：从学生真实痛点出发

创始人是大一学生，每天面对期末复习、课程论文、资料整理等真实场景。产品功能不是"想象学生需要什么"，而是"自己用、同学用、发现问题立刻改"。这是所有大公司和远距离创业团队不具备的**场景接近性**。

### 5.2 产品优势：成果导向而非聊天导向

市场主流 AI 学习产品 = 聊天界面 + AI 回复。MangoOS 的核心差异是**把聊天变成可交付物**：输入→检索→生成→审核→导出→沉淀。每个环节都可独立验证、独立优化。

### 5.3 内容优势：结构化 + 可交付 + 可复用

AI 生成内容不仅有正文，还有章节结构、引用来源、质量评分、版本历史。用户生成一次，永久保存。不是"问一次忘一次"的聊天模式。

### 5.4 增长优势：校园场景天然适合冷启动

- 期末复习 = 每年两次的硬刚需
- 学生之间天然有资料分享行为（PDF 讲义 = 天然传播载体）
- 第三自习室社群 = 种子用户池
- 小红书/微信群/朋友圈的内容传播形式天然适合"成品展示"获客

### 5.5 工程优势：快速迭代能力

- 单人全栈（Next.js + Supabase + DeepSeek）独立完成
- V1.0 到 V14.8.1 在数周内完成迭代，每周可交付功能性更新
- Build 100/100 页，0 TypeScript 错误的生产质量标准
- 不需要跨团队协调，决策→开发→部署链路极短

## 6. 目标用户

### 第一圈层：大学生（核心用户）
- **场景：** 期末复习讲义生成、课程资料整理、论文大纲、作业辅助
- **痛点：** 时间紧、资料散、不知道如何系统复习
- **付费意愿：** 期末场景付费意愿最高（按学期 / 按次）
- **触达方式：** 校园社群、第三自习室、期末资料分享

### 第二圈层：高中生 / 备考用户
- **场景：** 知识框架搭建、复习讲义、错题整理
- **痛点：** 缺乏系统性复习工具，AI 使用能力有限
- **触达方式：** 小红书、学习博主合作

### 第三圈层：留学生
- **场景：** 英文论文、报告整理、课程资料阅读
- **痛点：** 语言障碍 + 资料量大 + 格式要求高
- **触达方式：** 留学生社群、英文内容展示

### 第四圈层：内容生产者
- **场景：** 知识内容整理、结构化输出、付费内容制作
- **痛点：** 需要高质量、可交付的内容生成工具
- **触达方式：** 内容平台展示成品

### 第五圈层：教育工作者 / 辅导老师
- **场景：** 讲义生成、习题制作、学生学习分析
- **痛点：** 重复劳动多，需要 AI 辅助提效
- **触达方式：** B 端合作、教师社群

## 7. 市场机会

### 7.1 市场规模参考

> **注意：** 以下数据来源于公开市场研究报告，MangoOS 尚未进行独立的 TAM/SAM/SOM 测算。

- 据 HolonIQ 数据，全球教育科技市场预计在 2025 年达到约 4,040 亿美元，AI+ 教育是增速最快的细分领域之一。
- 据 Grand View Research，AI 教育市场 2023 年约 34.5 亿美元，预计 2024-2030 年复合年增长率（CAGR）达到 36.6%。
- 中国在线教育及教育科技市场在 2024 年进入理性发展期，AI 学习工具被视为新一轮增长驱动。
- **学生端 AI 学习工具市场仍处于早期，尚未出现明确的市场领导者。**

### 7.2 关键趋势

1. **AI 从聊天转向成果交付：** 用户不再满足于"AI 回答我一个问题"，而是希望"AI 帮我完成一个任务并交付成品"。
2. **个性化学习需求上升：** 标准化课程和统一进度越来越难以满足学生多样化需求。
3. **生成式 AI 成本持续下降：** DeepSeek 等中国模型的 API 价格使免费增值模式成为可能。
4. **校园 AI 工具窗口期：** 学生群体 AI 使用率在 2024-2025 年出现陡峭增长，但专用学习工具仍稀缺。

### 7.3 MangoOS 的切入时机

- 大模型推理成本已降至免费模式可承受范围
- 校园 AI 使用习惯正在形成，但尚未出现"学生 AI 学习操作系统"品类的强势品牌
- 本学期末复习场景是天然验证窗口

## 9. 增长策略

### 9.1 早期增长路径（0→100 用户）

**阶段 1：校园内测**
- 第三自习室社群作为种子用户池
- 同学、室友、课程小组 → 真实任务测试
- 收集 20-50 个真实学习任务案例

**阶段 2：期末复习刚需切入**
- 期末考试前 2-3 周集中推广
- 生成"XX 大学 XX 课程期末复习讲义" → 自然传播
- PDF 讲义文件名和页脚含 MangoOS 品牌 → 二次传播

**阶段 3：成品展示获客**
- 小红书 / 微信群 / 朋友圈展示用 MangoOS 生成的学习成果
- "用 AI 整理了一门课的期末讲义，18 个章节带 PDF"→ 好奇心驱动注册
- 不做广告式宣传，做成果式展示

### 9.2 留存策略

- 历史成果可复用 → 用户生成过一门课的资料后会回来生成第二门
- 闪卡 SM-2 算法 → 每日复习提醒 → 形成使用习惯
- Pro 功能体验 → 免费用户可预览 Pro 输出质量（只读）→ 转化动机

### 9.3 付费转化策略

- 期末场景天然转化窗口：免费生成预览 → 导出 PDF 需 Pro
- 体验式转化：生成过 3 次免费任务 → 弹出 Pro 试用（7 天）
- 校园团购：室友 / 班级一起买，降低个体决策门槛

### 10.2 技术指标

- **构建：** 100/100 页面，0 TypeScript 错误
- **路由：** 30+ 页面路由，24 组 API 路由
- **数据库：** 26 张 Supabase 表，全部 RLS 保护
- **部署：** Vercel 自动部署（mangoleaningos.top）
- **AI 引擎：** DeepSeek（可插拔，OpenAI 兼容协议）
- **字体：** 自托管（零外部依赖，中国网络可用）

### 10.3 当前问题（诚实列出）

1. **功能过多，主线不清晰。** 30+ 页面涵盖学习、心理、计划、知识管理、语音等，投资人可能看不懂核心价值。
2. **Agent 输出质量仍不稳定。** 质量门禁已上线但取决于 AI 模型能力，部分低分输出会自动标记 FAILED 但无法 100% 修复。
3. **UI 仍有"AI 味"。** 部分页面设计语言不够统一，需要进一步去工具化、增强沉浸感。
4. **真实用户验证不足。** 功能开发和自测为主，尚未规模化收集真实学生用户的使用数据。
5. **付费验证为零。** 芒果码机制存在但未进行真实付费实验。
6. **创始人时间瓶颈。** 一人全栈开发的可持续性问题。

### 10.4 近期策略

- **暂停高强度功能开发**
- **优先完成期末场景成果闭环验证**
- **收集真实使用案例和用户反馈**
- **精简功能展示，突出核心差异**
- **维持低频技术维护，保 build 绿**
