## 1. Tailwind CSS v4 — Accessible Color Contrast

### Reference: Tailwind CSS Docs — Dark Mode & Custom Variants
- **URL**: https://tailwindcss.com/docs/dark-mode
- Tailwind v4 uses `@custom-variant dark (&:is(.dark *))` — already adopted in MangoOS

**Key Pattern**: Opacity modifiers (`/60`, `/40`) on already-muted colors compound to produce contrast below WCAG AA (4.5:1). The fix: reduce max opacity drop to `/80` minimum for body text, and use dedicated `text-fg-subtle` for tertiary text instead of `text-fg-muted/50`.

**MangoOS Adaptation**: Bulk-replaced `text-fg-muted/30→/40→/50→/60` with safer equivalents (`/80` minimum for muted, `/90` for subtle). Applied across 33 files via script.

**Verdict**: ADAPT ✅

## 7. Mobile Readability — Contrast & Font Usage

### Reference: WCAG 2.1 AA Contrast Guidelines
- Minimum contrast: 4.5:1 for normal text, 3:1 for large text (≥18px bold or ≥24px)
- Low-opacity text on cards is the #1 readability failure in production apps

**Key Finding**: MangoOS had systematic `/30`, `/40`, `/50`, `/60` opacity modifiers on `text-fg-muted` and `text-white` creating contrast below 2:1 in many cases. Also, `font-serif` (Cormorant Garamond) was used for body text where it's harder to read at small sizes.

**MangoOS Adaptation**:
1. Bulk-replaced low-opacity text patterns across 33 files
2. Kept `font-serif` only for headings (`.text-display.font-serif`, `.font-serif` on titles)
3. Body text, buttons, nav, descriptions all use `font-sans` (Inter)

**Verdict**: ADAPT ✅

## 8. Mobile Shell — Glass Card Contrast

### Issue Found
`.mango-glass-card` uses `rgba(255,255,255,0.115)` background with `rgba(255,255,255,0.045)` gradient. Text at `text-white/40` on this background is nearly invisible.

### Fix Applied
Source cards and action bars on glass backgrounds now use `text-white/80` minimum. Research source cards use `bg-bg-subtle` (solid) instead of glass for readability.
