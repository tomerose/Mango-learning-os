# COMPETITOR_RESEARCH.md — 竞品研究：他们为什么让用户持续使用

**日期:** 2026-06-05 | **研究方法:** Web Search + 产品分析 + 留存框架

---

## 1. 学习产品

### 1.1 Duolingo — 习惯机器的标杆

**数据:** 500M+ 用户, 20M+ DAU, $12B 估值

**核心留存机制:**

| 机制 | 心理学原理 | 数据效果 |
|------|-----------|---------|
| Streak 连续天数 | 损失厌恶 — 断签=真实损失感 | Day-14 留存 +14% |
| 勋章 & 成就 | 社交证明 + 可分享里程碑 | 推荐率 +116% |
| 周联赛 | 软竞争 + 公平匹配 | 动机衰减时续命 |
| 即时反馈 | 多巴胺循环 (XP/动画/音效) | 维持心流 |
| 冷冻道具 | 减少焦虑 → 反而增加长期参与 | "增加第二个冷冻后DAU暴涨" |
| 吉祥物(Duo) | 友好人格 → 通知从"骚扰"变"提醒" | DAU +5% |

**对Mango的启示:**
- 连续学习天数(streak)是Mango已有的资产，但它没有和每日任务关联
- Duolingo的"5分钟完成今日任务"是Mango缺少的：**"我今天的5分钟是什么？"**

### 1.2 Quizlet — AI 闪卡之王

**核心价值:** 用户输入→AI自动生成闪卡/测验/游戏

**为什么用户每天打开:**
- Flashcard是最低摩擦的学习行为（30秒完成一轮）
- AI自动从笔记/文档生成卡片（输入→输出闭环）
- 学习模式多样性（Learn/Test/Match/Gravity）

**对Mango的启示:**
- Mango有SM-2算法但没有自动卡片生成
- Mango的闪卡藏在exam→第3个tab，Quizlet直接放在首屏

### 1.3 Khanmigo (Khan Academy × GPT-4)

**核心价值:** 苏格拉底式AI导师，不直接给答案

**关键设计:**
- AI永远不会直接给出答案 — 它反问你、引导你、检查你的推理
- 每个对话都有明确的学习目标（不是自由聊天）
- 教师仪表盘：老师可以看到每个学生的AI对话摘要

**对Mango的启示:**
- Mango的Agent应该从"回答机器"变为"引导者"
- 每次对话应该有明确学习目标而非自由聊天

### 1.4 NotebookLM (Google)

**核心价值:** "上传你的资料 → AI变成你的私人研究助理"

**关键设计:**
- 用户上传自己的资料（PDF/文档/笔记）
- AI只在用户提供的资料范围内回答 — 不编造
- **Audio Overview**: 把资料变成双人对话播客

**对Mango的启示:**
- Mango的Magic功能没有"基于我的资料"这个约束
- 上传资料 → AI生成所有学习材料 → 这才是完整的内容生成闭环

### 1.5 Perplexity

**核心价值:** AI搜索 + 来源引用 + 追问

**为什么日常使用:**
- 每次搜索都有来源链接（信任）
- 自动生成追问建议（探索）
- Pro Search模式进行多步推理

**对Mango的启示:**
- Mango的所有AI输出都应标注来源（或标注"由你的XX资料生成"）
- 生成后的追问建议 = 学习深入的关键

## 2. AI Agent / 开发工具

### 2.1 Cursor — AI原生IDE

**数据:** SaaS历史上增速最快的创业公司之一

**核心设计哲学:**

| 原则 | 实现 |
|------|------|
| AI是环境而非工具 | 集成到文件树/终端/Linter/编辑器 |
| 速度保护心流 | 自研Composer模型，4倍速，<30s完成turn |
| 渐进式投入 | Tab→Inline Edit→Agent→Plan Mode→Multi-Agent→Cloud Agent |
| 人是审阅者而非作者 | Diff审批/Plan审批/聚合Review/Test-First 循环 |
| 治理建立信任 | Team Rules/Hooks/索引上下文范围 |
| 并行而非串行等待 | 最多8个Agent并行，Compare & Merge |

**为什么开发者每天打开:**
- 无论有30秒还是3小时，都有一个适配的AI交互模式
- Cmd+K + 选中代码 + 描述修改 → 即时diff → 这是"5秒快感"
- Agent负责脚手架，人负责review → 工作流融入日常

**对Mango的启示(极重要):**
- Mango需要"5秒快感"：打开→看到今日任务→完成一个→看到反馈
- Mango需要"渐进式AI"：从"做一道题"到"生成完整复习计划"
- 学习Agent应该像Cursor的编程Agent一样，有Plan→Build→Review循环

### 2.2 Linear — 项目管理标杆

**为什么设计师和工程师日常使用:**
- 键盘优先：不用鼠标即可完成所有操作
- 速度是feature：每个交互<100ms
- 视觉克制：没有多余UI元素
- 明确的层级：Issue→Project→Cycle

**对Mango的启示:**
- Hub页应该像Linear的inbox：一眼看到"今天要做什么"
- 学习任务应该有明确的"完成"信号

### 2.3 Claude Code / Claude

**为什么日常使用:**
- 规划模式：先plan→review→build
- Memory系统：记住用户偏好
- 文件直接操作：不是聊天工具，是执行工具

**对Mango的启示:**
- Mango的Agent应该能执行操作（创建任务/调整计划）而不只是聊天
- 应该有Plan模式：用户说目标 → Agent生成计划 → 用户审批 → Agent执行

## 3. 知识管理产品

### 3.1 Notion — 数字化思考空间

**为什么成为日常习惯:**
- 成为"默认思考场所"：打开就开始写
- AI集成：选中文字 → "Ask AI"直接处理
- 数据库视图：同一数据多种可视化

**对Mango的启示:**
- 学习笔记不应该只是"保存文本"，应该能变成闪卡/测验/复盘资料
- 同一个学习内容应该有多种"视图"

### 3.2 Capacities — 基于对象的知识管理

**设计哲学:** 不是文件/文件夹，而是"对象"（人、书、项目、想法）

**对Mango的启示:**
- 学习内容可以按"对象"组织：概念→相关的题目→相关的笔记→相关的项目
- 知识图谱应该反映真实的关联而非AI生成的静态图

## 4. 留存框架：他们为什么让用户每天打开

### 4.1 Hook Model (Nir Eyal)

```
TRIGGER (可靠且上下文相关)
  → ACTION (低动机状态也能完成的最简行为)
    → REWARD (满足需求 + 创造期待)
      → INVESTMENT (数据/身份/关系)
```

### 4.2 各产品的Hook对比

| 产品 | Trigger | Action | Reward | Investment |
|------|---------|--------|--------|------------|
| Duolingo | 通知(吉祥物) + 内部("我的streak!") | 5分钟一节课 | XP/宝石/联赛升级 | 200天streak不能断 |
| Anki | 每日due cards提醒 | 翻一张卡(5秒) | "我记得!"的满足感 | 卡片库=知识资产 |
| Notion | 内部("我要记下来") | 打开新页面开始写 | 文档成形满足感 | 整个知识库在Notion |
| Cursor | 内部("要写代码了") | Cmd+K改一行 | 看到diff一键接受 | 项目在Cursor里成长 |
| Perplexity | 外部(搜索需求) | 输入问题 | 来源链接+追问 | 搜索历史=研究轨迹 |
| Linear | 内部("今天要做什么") | 快捷键过任务 | 任务完成动画 | Sprint进度可视化 |

### 4.3 Mango当前Hook缺口

```
TRIGGER: 无。用户没有内部触发（"我的streak!"）也没有可靠外部触发
ACTION:  无。打开后没有明确的"5秒行为"
REWARD:  弱。XP增长不可视、学习成果不可量化
INVEST:  弱。没有累积的知识资产展示、项目没有进度
```

## 5. 关键发现总结

### 🔴 Mango最缺失的5件事

1. **"我今天要做什么？"** — 打开app后没有答案
2. **"5秒快感"** — 没有一个极低摩擦的入口行为
3. **"我的知识资产"** — 笔记/闪卡/反思散落在7个窗口中，看不到累积
4. **"AI知道我吗"** — Agent Memory存在但不被感知
5. **"我进步了吗"** — 没有能力成长的可视化

### 🟢 竞品已证明有效的模式

1. **目标驱动 > 功能选择** — Khanmigo每次对话有目标，Cursor每次任务有Plan
2. **AI作为环境 > AI作为工具** — NotebookLM基于用户资料，Cursor集成到IDE
3. **渐进式AI** — 从简单到复杂，从手动到自动
4. **内化触发** — 用户打开app是因为"我需要"，不是"我收到了通知"
5. **知识→行动闭环** — Quizlet从笔记生成闪卡，从闪卡生成测验，从测验发现薄弱
