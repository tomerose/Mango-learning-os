## 1. 产品愿景 (Product Vision)

**AI Learning OS 是一个把学习科学设为默认交互、让 AI 当陪练而非代笔、跨学科统一调度的个人学习操作系统。** 它对抗一个生物学默认值——遗忘——通过把主动回忆、间隔重复与交错练习从"可选功能"变成"强制底层交互"，让逻辑驱动、时间稀缺的跨学科学习者（以金融 AI 学生为原型）把碎片课程知识沉淀为可检索、可复用、考完不忘的体系。它衡量的唯一标准不是"你停留了多久"，而是"你真的学会了什么"：用提取成功率与知识迁移作为北极星，用诚实的双轨进度让胜任感自我兑现，用学期节奏感知在考试周自动重排战线优先级。这不是又一个内容仓库或聊天助手，而是一个统一调度"AI + 量化 + 英语 + 数学"四条战线、把循证学习方法默认烧进每一次交互的个人学习操作系统。

## 2. 核心价值主张 (Core Value Propositions)

围绕研究锁定的不可替代价值 (N1–N4) 与架构性差异化 (D1–D6)，提炼 3 条最高优先价值：

### CVP1 — 学习科学是默认，不是选项 (Science by Default)
对标 Notion / Coursera (D1, G1)。竞品允许用户继续用"流畅但无效"的重读/高亮 (LS#2)；本产品把闭卷自测、扩展间隔复习、交错出题设为默认交互，并用**提取成功率**而非浏览时长作为掌握度信号。难以模仿的原因：这要求重构核心交互模型，与"内容消费"产品哲学根本冲突。

### CVP2 — AI 是陪练，不是代笔 (Coach, not Ghostwriter)
对标 ChatGPT / Claude (D2, G2)。主流 AI 默认直接给答案，抹平合意困难、助长认知卸载 (LS#10)。本产品用结构化提示让 AI 先反问、挑战、要求用户先产出再校验——把 AI 从"削弱思考"变成"强制元认知参与"。这直击金融 AI 学生"既要用 AI 又怕被 AI 削弱"的核心矛盾 (N4)。

### CVP3 — 跨学科统一调度 + 学期节奏感知 (One OS for Four Fronts)
对标 Anki / Duolingo 单领域工具 (D3, G4/G5)。没有产品能跨"AI+量化+英语+数学"四线做优先级仲裁，也没有产品能感知考试周自动收缩自学、聚焦提分 (N5/N6)。这是把分散的单点工具升维成"个人学习 OS"的关键，单领域工具因路径依赖无法横向扩展。

## 3. MVP 功能清单 (MVP Feature List)

按 `Function > Utility > UX > Scale` 分层，并把用户指定的六大模块 (Dashboard, AI Tutor, Study Planner, Knowledge Hub, Exam Mode, Profile) 与四大中心 (University Exam Assistant, AI Career Growth Center, Personal Growth Center, Knowledge Management System) 全部归类。

### P0 — MUST HAVE (上线临界，不做就没有产品)

| 模块 / 功能 | 说明 | 命中 |
|---|---|---|
| **Dashboard (仪表盘)** | 双轨进度可视化 (F4)：累计成就 vs 当前目标；自我参照（对比过去的你）；每个图表回答"下一步做什么"。今日复习队列 + 待办优先级入口。 | N2 / DP2 / BP§5.1#5.2 |
| **AI Tutor (AI 陪练)** | 苏格拉底式陪练 (F3)：AI 先反问/挑战，用户先产出后校验，拒绝直接代写。逻辑优先 + 结构化 + 双语呈现 (D4)。 | N4 / LS#10 / G2 |
| **Knowledge Hub / KMS (知识中枢 + 知识管理系统)** | 知识捕获 → 自测卡片转化 (F2)：笔记/收藏一键转提取式卡片，捕获即进复习队列；对抗囤积 (G3)。这是 KMS 的 MVP 形态。 | N1#3 / LS#2 / G3 |
| **间隔重复引擎 (SRS Engine)** | 主动回忆 + 扩展间隔自动排程 (F1)；闭卷自测为默认交互。系统底层能力，驱动 Dashboard 与 Knowledge Hub。 | N1 / LS#1#3#4 / G1 |
| **Study Planner (学习计划) — 基础版** | 把复习队列与今日任务排进时间；支持碎片时间轻量模式入口 (F8 的 MVP 切片)。 | N6 / BP§3.3 |
| **Exam Mode / University Exam Assistant (考试模式 + 大学考试助手)** | 学期节奏 / 考试周感知调度 (F6)：感知考试临近自动收缩自学、聚焦提分；分散复习反突击。这是 Exam Assistant 的 MVP 核心。 | N5 / G4 / LS#4 |
| **Profile (个人档案) — 基础版** | 学习目标设定、四线领域配置、隐私基线（数据可导出/删除）。陪练与调度的个性化依赖此。 | DP8 / BP§5.3 |

**MVP 最小闭环**：SRS 引擎 + Knowledge Hub(F2) + AI Tutor(F3) + Dashboard(F4) = "捕获知识 → 转自测 → AI 陪练校验 → 间隔复习 → 看见真实进步"，首次会话内完成一次完整闭环 (BP§1 aha moment / TTV)。

|---|---|---|

| **Study Planner — 跨任务优先级仲裁 (F10 切片)** | 四线统一调度雏形：高优先级抢占、低优先级分解延后。从单领域起步。 | N6 / P§3.1 / A4 |

|---|---|---|

## 4. 核心用户流 (Top 5 User Flows)

### Flow 1 — 首次会话学习闭环 (Aha Moment / TTV)
新用户注册（极简，注册前可体验）→ 选择一个领域与一个目标 → 粘贴/导入一段课程笔记 → 系统一键转为 3–5 张提取式自测卡 → 立即闭卷自测一轮 → AI 陪练对错题反问挑战、要求复述 → 卡片进入间隔复习队列 → Dashboard 首次点亮"今天你真的学会了 X"。**目标：5 分钟内完成首次完整闭环 (DP5)。**

### Flow 2 — 每日复习循环 (Daily Active Loop)
进入 App（锚定午餐/睡前触发点）→ Dashboard 显示今日到期复习队列 → 进入轻量模式 → 闭卷自测到期卡片 → 标记回忆质量 → SRS 按表现重排下次间隔 → 难点卡自动转 AI 陪练深挖 → 连胜推进（断签可冻结/补签）→ 双轨进度更新"对比过去的你"。**目标：核心循环 ≤5 分钟 (F8/N6)。**

### Flow 3 — AI 苏格拉底陪练 (Coach, not Ghostwriter)
用户提出问题或卡在某概念 → AI **不直接给答案**，先反问"你目前的理解是什么" → 用户先产出 → AI 挑战/指出漏洞/给提示 → 用户修正 → AI 用 worked example 或类比补足 (DP6) → 最后让用户用自己的话复述并存为自测卡。**强制元认知参与 (D2/N4)。**

### Flow 4 — 考试周自适应重排 (Exam Mode Trigger)
用户在 Profile/Planner 录入考试日期 → 系统感知考试临近 → 自动收缩自学战线、聚焦该科提分 → 把分散复习前移、压制突击诱惑 → 生成考前冲刺计划（含交错练习 F5）→ 可选启用焦虑模块（表达性写作 + 睡眠提醒）→ 考后自动恢复常态调度。**学期节奏感知 (N5/G4/F6)。**

### Flow 5 — 知识捕获与沉淀 (Anti-Hoarding Capture)
用户遇到值得保留的信息（文章/课堂/AI 对话）→ 一键捕获到 Knowledge Hub → 系统提示"要内化还是仅归档" → 选择内化则即时转为自测卡并入复习队列 → 打跨学科标签便于检索与交错 → 定期"囤积体检"提示长期未复习的沉睡卡片。**捕获即复习，囤积≠内化 (N1#3/G3/F2)。**

## 6. 6 个月路线图 (6-Month Roadmap)

### MVP (Month 1–3) — 跑通核心闭环
**目标**：验证 N1–N4 与最小闭环 TTV。范围严格锁 P0。
- M1：SRS 引擎 + Knowledge Hub(F2 捕获转卡) + Profile 基础 + 极简引导。
- M2：AI Tutor 苏格拉底陪练(F3) + Dashboard 双轨进度(F4)。
- M3：Study Planner 基础排程 + Exam Mode 学期感知(F6) + 首次闭环打磨至 ≤5 分钟。
- **里程碑出口标准**：新用户首次会话完成完整闭环 (Flow 1)；A2（合意困难 vs 留存张力）有初步实测数据。**单领域先行（数学或英语）以控范围 (A4)。**

### V1.1 (Month 4–5) — 留存与可持续
**目标**：提升 D1/D7（留存）。范围放开 P1。
- 交错出题引擎(F5) + 连胜宽容机制(F7) + 碎片时间轻量模式完整版(F8)。
- Study Planner 跨任务优先级仲裁切片(F10) + AI Career Growth Center 基础版。
- 引导期降难度、核心期上难度的自适应策略（回应 A2）。
- **里程碑出口标准**：D7/D30 留存达标；连胜与轻量模式显著降低断签流失。

### V1.2 (Month 6) — 增强、合规与横向扩展
**目标**：补合规底线 + 验证横向扩展。范围引入关键 P2。
- 焦虑与睡眠模块(F9) + 无障碍/隐私完整版(F11 / WCAG AA)。
- Personal Growth Center 入口 + 从单领域扩到第二条战线（验证跨学科调度）。
- 收集 A1（用户群规模/付费意愿）验证数据，为商业化与 P3 决策铺垫。
- **里程碑出口标准**：WCAG AA 合规；第二战线接入不破坏核心闭环；A1/A4 验证结论产出。

## 7. 成功指标 (Success Metrics / KPIs)

北极星：**用户真的学会了什么**（提取成功率 / 知识迁移），而非停留时长 (DP7)。所有指标服从此红线，拒绝成瘾式时长指标。

### 7.1 学习成果 (Learning Outcomes) — 北极星层
- **提取成功率 (Retrieval Success Rate)**：到期卡片闭卷自测的正确率趋势（关键：随间隔扩展仍保持高位）。
- **长期保持率 (Long-term Retention)**：捕获后 30/60 天的卡片提取正确率（直接衡量"对抗遗忘"）。
- **知识迁移 (Transfer)**：交错/新情境题正确率 vs 单一练习题，衡量真实掌握而非流畅性错觉。
- **内化转化率 (Internalization Rate)**：捕获内容中"转为自测卡并完成至少一轮复习"的比例（对抗囤积）。

### 7.2 参与度 (Engagement)
- **首次闭环完成率 (TTV / Aha Rate)**：新用户首次会话内完成 Flow 1 的比例（目标导向，非时长）。
- **每日核心循环完成率**：进入 App 的用户完成当日复习队列的比例。
- **AI 陪练元认知参与率**：陪练会话中"用户先产出后校验"的比例（衡量 D2 是否真正落地，非单纯调用次数）。
- **每会话有效学习动作数**：自测/复述/校验等"合意困难"动作数（替代"停留时长"）。

### 7.3 留存 (Retention)
- **D1 / D7 / D30 留存率**：习惯地基是否成立。
- **复习队列按时清空率**：衡量间隔调度被坚持的程度。
- **断签恢复率**：使用冻结/补签后回归的比例（衡量 F7 宽容机制有效性）。
- **考试周后留存**：考试模式触发后，考后是否回归常态调度（衡量学期节奏感知的黏性）。

### 7.4 健康 / 伦理护栏 (Health Guardrails)
- **诚实进度信任度**：用户对进度展示真实性的反馈（守住 DP7，拒绝虚假"你真棒"）。
- **过载预警**：单日任务超载导致的流失监控（认知负荷感知 DP6）。
- **无障碍达标率**：WCAG AA 关键路径覆盖（DP8）。

## 8. 关键风险与裁决 (Risks & Decisions)

承接 SYNTHESIS §7 的开放假设，落到产品决策：
- **A1 单画像风险** → V1.2 收集用户群/付费验证数据再做商业化决策。
- **A2 合意困难 vs 留存张力** → 采纳"引导期降难度、核心期上难度"，MVP 出口必须有实测。
- **A3 AI 陪练落地难度** → F3 是技术+产品双重风险，MVP 优先打磨提示工程质量，用"元认知参与率"而非调用量度量成败。
- **A4 范围蔓延** → 跨学科仲裁(F10)与第二战线推迟，MVP 单领域先行。
- **A5 效应量保守** → 学习成果 KPI 目标值设保守基线，避免过度承诺。

---

> 决策摘要：MVP 锁 P0 七模块、跑通最小闭环、单领域先行；V1.1 攻留存；V1.2 补合规并验证横向扩展。一切服从北极星——衡量"学会"而非"停留"。
