# V14.7 GitHub Reference Notes

## Inspected Projects

### 1. shadcn/ui (ui.shadcn.com)
- **Files**: card, button, dialog, input, textarea, skeleton, sheet components
- **Useful**: composable component API, slot-based composition, variant system
- **Adapted**: card/skeleton/button patterns for outcome document layout
- **Rejected**: full dashboard templates (too marketing-heavy for MangoOS)

### 2. Vercel AI SDK (sdk.vercel.ai)
- **Useful**: useChat streaming, tool-call UI pattern, App Router route design
- **Adapted**: streaming result display pattern (not full SDK, just UX concept)
- **Rejected**: useChat hook directly (MangoOS uses custom DeepSeek client)

### 3. TanStack Query (tanstack.com/query)
- **Useful**: loading/error/retry state machine, stale-while-revalidate
- **Adapted**: state pattern for file read/save/export (manual implementation, no dependency)
- **Maps to MangoOS**: library fetch, artifact save, export states
- **Rejected**: adding the dependency (project uses manual fetch, adding TanStack Query now would be risky)

### 4. Zustand persist middleware
- **Useful**: localStorage hydration, partialize, version migration
- **Adapted**: intent persistence pattern (already in MangoTodayEntry)
- **Rejected**: replacing existing useStore (too risky, not needed)

### 5. React Hook Form (react-hook-form.com)
- **Useful**: form validation, file input handling, error display
- **Adapted**: validation pattern only (manual implementation for file upload)
- **Rejected**: adding the dependency (forms are simple enough)

### 6. FileReader API patterns (MDN + various GitHub examples)
- **Useful**: readAsText, readAsArrayBuffer, File.type checking
- **Adapted**: lib/file/file-reader.ts — Promise-based FileReader wrapper
- **Maps to MangoOS**: Agent file upload fix

### 7. Next.js App Router streaming (official docs)
- **Useful**: ReadableStream, AI route patterns
- **Adapted**: already used in Agent execute route
- **Status**: working, no changes needed

### 8. react-pdf / @react-pdf/renderer
- **Useful**: PDF generation patterns
- **Rejected**: too heavy for current needs, print-to-PDF is sufficient for V14.7
- **Future**: P2 consideration

## Adaptation Summary

| Pattern | Source | Adapted To |
|---------|--------|------------|
| Promise-based FileReader | MDN/Web APIs | lib/file/file-reader.ts |
| Structured intent routing | Zustand persist pattern | lib/today/intent-router.ts |
