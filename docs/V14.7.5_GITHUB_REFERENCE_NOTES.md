## 1. @react-pdf/renderer — PDF Generation
**Repo**: [diegomura/react-pdf](https://github.com/diegomura/react-pdf) (16.6k★)
**Key Pattern**: 
- `Document → Page → View/Text` component tree
- `StyleSheet.create()` CSS-in-JS
- `Font.register()` TTF fonts
- 4 export strategies: browser (`<PDFViewer>`), browser-download (`usePDF().toBlob()`), Node stream (`renderToStream`), Node buffer (`renderToBuffer`)

**MangoOS**: Browser-print approach selected for V14.7.5 — zero dependency, instant deployment. Full @react-pdf/renderer integration deferred to V14.8+ (adds ~200KB bundle).

**Verdict**: ADAPT-WITH-MOD (browser print now, server-side PDF later)

## 2. shadcn/ui Alert + Card Patterns
**Repo**: [shadcn-ui/ui](https://github.com/shadcn-ui/ui) (115k★)
**Key Patterns**:
- Alert: compound component (`Alert → AlertTitle + AlertDescription + AlertAction`), variant="destructive"
- Card: `Card → CardHeader(CardTitle+CardDescription) → CardContent → CardFooter`, `--card-spacing` CSS variable
- Image Card: `has-[>img:first-child]:pt-0` pattern for edge-to-edge

**MangoOS**: Quality V3 breakdown card uses card-card pattern; Alert destructive pattern used for gate failures. Already aligned.

**Verdict**: ADAPT ✅ (in use)

## 3. Vercel AI SDK — Tool-Call UI & Generation Lifecycle
**Repo**: [vercel/ai](https://github.com/vercel/ai) (24.7k★)
**Key Patterns**:
- Tool part state machine: `input-streaming → input-available → output-available / output-error`
- `step-start` boundary markers between sequential stages
- `dynamic-tool` type for runtime-loaded tools
- `onToolCall` client-side auto-execution

**MangoOS**: Agent view state machine (templates/running/result/tasks) mirrors the status enum pattern. Timeline events map to `step-start` boundaries. Auto-deepen uses tool-call re-invocation pattern.

**Verdict**: ADAPT (conceptual alignment) ✅

## 4. Research-Agent Citation UI Patterns
**References**: genai-explorer, deep-agents, olympus (Perplexity-style)
**Key Pattern** — 3-tier progressive disclosure:
1. **Level 1 (Summary)**: Inline citation badges `[1]` `[2]` — always visible, clickable
2. **Level 2 (Detailed)**: Expandable source cards with domain, snippet, relevance score
3. **Level 3 (Technical)**: Full trace logs — behind advanced toggle

**Citation Data Model**: `{ index, source_id, title, url, domain, snippet, relevance }`

**MangoOS**: Source cards in result view (Level 2). Inline badge parsing `[1] [2]` not yet automated — next priority.

**Verdict**: ADAPT-WITH-MOD (Level 2 implemented, Level 1 parsing deferred)
