## 0. 核心论点 (Thesis)

三份研究在同一处交汇：**遗忘是默认的生物过程，绝大多数学习失败源于"策略错配"与"认知架构错配"——而现有工具几乎没有把已被反复验证的循证方法（主动回忆 + 间隔重复 + 交错练习）设为默认强制交互。**

- 学习科学告诉我们 **什么有效**（合意困难 / desirable difficulties）。
- 用户画像告诉我们 **为谁而建**（林深：逻辑驱动、时间稀缺、跨学科、工具敏感的金融 AI 大一生）。
- 最佳实践告诉我们 **如何让它被坚持下来**（SDT 内在动机 + 习惯地基 + 可行动分析）。

三者的共同北极星：**衡量成功的唯一标准是"用户真的学会了什么"，而非"用户停留了多久"** (BP§0/§3.4)。

## 1. 关键用户需求 (Critical User Needs)

把用户画像的 4 类痛点 (P§3) 与学习科学的根因 (LS) 对齐后，浮现 6 项**深层需求**。每项标注证据三角（画像痛点 × 学习科学机制 × 最佳实践对策）。

| 编号 | 关键需求 | 画像痛点 | 学习科学根因 | 最佳实践支撑 |
|---|---|---|---|---|
| **N1** | **对抗遗忘的知识沉淀**：把碎片课程知识转为可检索、可复用、不会考完即忘的体系 | P§3.3 信息囤积；P§2.1 核心诉求 | LS#1 遗忘曲线 / LS#2 流畅性错觉 / LS#3 未用主动回忆 | BP§3.1 投入累积成切换成本 |
| **N2** | **即时可验证反馈维持动机**：自学缺乏反馈，看不到进步就掉链子 | P§3.4 动机波动 | LS-D 自我效能互为因果；LS-E 反馈效应 d≈0.73 | BP§4.1 胜任感；BP§5.2 自我参照进步 |
| **N3** | **信号过滤 + 防囤积**：AI 领域信息过载，收藏夹吃灰，分不清信号与噪音 | P§3.3 信息过载 | LS#8 认知超载（工作记忆 ~4 项） | BP§0 价值优先于花哨；BP§5.1 可行动性 |
| **N4** | **AI 陪练而非代笔**：作为金融 AI 学生，既要用 AI 又怕被 AI 削弱思考 | P§2.2 AI 职业能力 | LS#10 AI 认知卸载削弱批判性思维 | BP§4.2 内化目标价值 (OIT) |
| **N5** | **考试周焦虑管理**：期中/期末与自学冲突，推导焦虑吞噬工作记忆 | P§3.2 考试压力 | LS#5 数学焦虑占用工作记忆；LS-C 睡眠 | BP§4.1 自适应难度维持心流 |
| **N6** | **跨任务优先级仲裁 + 碎片时间利用**：四条战线抢时间，"什么都想做，什么都没做完" | P§3.1 时间管理 | LS#6 拖延（自我调节失败）；LS#7 数字分心 | BP§3.1 锚定既有习惯；BP§3.3 低单次门槛 |

**洞察**：N1–N4 是产品的**不可替代价值**（直击学习成败的核心机制）；N5–N6 是**留存与可持续性**的保障（让用户能长期用下去）。MVP 必须先解决 N1–N4。

## 2. 市场空白 (Market Gaps)

竞品按"做对了什么 / 缺了什么"拆解。空白 = 现有工具的系统性缺失，正是本产品的机会窗口。

| 竞品类别 | 代表 | 做对了 | 系统性缺失（=我们的空白） |
|---|---|---|---|
| SRS 闪卡 | Anki | 间隔重复引擎强 (LS#1) | 无内容生成、无结构、无 AI、学习曲线陡、跨学科组织差 |
| 游戏化语言 App | Duolingo | 习惯/连胜机制业界标杆 (BP§2.2) | 单一领域；动机建在外部奖励，缺 SDT 内化 (BP§4.2)；浅层 |
| 知识管理 | Notion / Obsidian | 笔记沉淀、可检索 | **被动仓库**：不主动调度复习，违背遗忘曲线 (LS#1)；无主动回忆强制 |
| 通用 AI 助手 | ChatGPT / Claude | 强大、即时 | **助长被动卸载** (LS#10)：默认直接给答案，抹平 desirable difficulties，无元认知强制 |
| 传统 LMS | Coursera 类 | 内容体系完整 | 被动观看为主 (LS#2)；无个性化间隔调度；无优先级仲裁 |

**5 大空白（按价值排序）**：

- **G1 — 循证方法未被默认强制**：没有产品把"主动回忆 + 间隔重复 + 交错练习"设为**默认交互**而非可选功能。用户被允许继续用无效的重读/高亮 (LS#2)。
- **G2 — AI 助长卸载而非参与**：主流 AI 工具默认"代笔"，没有产品在架构上**强制元认知参与**（先产出后校验、苏格拉底式反问）(LS#10)。
- **G3 — 知识仓库不会主动复习**：笔记类工具是静态存储，**不按遗忘曲线主动调度** (LS#1/#4)，囤积≠内化 (P§3.3)。
- **G4 — 缺学期节奏感知**：没有产品**感知考试周并自动重排**学习计划（考试周收缩自学、聚焦提分）(P§3.2 / 设计启示)。
- **G5 — 缺跨学科统一 OS**：单点工具各管一摊，没有**跨任务优先级仲裁**的 Personal OS 来调度"AI+量化+英语+数学"四线 (P§3.1)。

## 3. 差异化竞争优势 (Key Differentiators)

我们不靠"功能更多"取胜，靠**架构性选择**取胜——这些选择竞品因路径依赖难以模仿。

| 编号 | 差异化 | 对标谁 | 为什么难被模仿 |
|---|---|---|---|
| **D1** | **学习科学是默认交互，非可选项**：自测/提取为默认；用"提取成功率"而非"浏览时长"作掌握度信号 (LS#2 流畅性错觉防护) | Notion / Coursera | 需重构核心交互模型，与"内容消费"产品哲学冲突 |
| **D2** | **AI 陪练模式**：结构化提示让 AI 反问、挑战用户；先自己产出再用 AI 校验 (LS#10) | ChatGPT | 与"即时给答案"的 AI 产品默认体验背道而驰 |
| **D3** | **跨学科统一 Personal OS**：单一系统仲裁四条战线优先级 + 学期节奏感知 (P§3.1/§3.2) | Anki / Duolingo | 单领域工具无法横向扩展到优先级仲裁 |
| **D4** | **逻辑优先 + 结构化 + 双语呈现**：概念→直觉→推导→例子→陷阱→应用；中文默认 + 英文术语附释义；表格/流程图为一等公民 (P§6) | 通用工具 | 深度契合精准画像，泛化产品不会为此优化 |
| **D5** | **MCP-first / workflow-first 架构**：复用基础设施，工作流优于单点工具 (P§5.3) | 所有竞品 | 契合 power user，主流产品面向大众做 GUI 一键式 |
| **D6** | **伦理参与：衡量"学会"而非"停留"**：拒绝成瘾设计，进度数据诚实 (BP§3.4/§5.2) | Duolingo 类 | 与广告/时长驱动的商业模式冲突 |

**一句话定位**：*"一个把学习科学设为默认、让 AI 当陪练而非代笔、跨学科统一调度的个人学习操作系统。"*

## 4. MVP 必备功能 (Must-Have Features)

按用户价值序 **Function > Utility > UX > Scale** 分层。P0 = 不做就没有产品；P1 = 留存关键；P2 = 增强与合规。

### P0 — 核心价值（直击 N1–N4，对应 G1–G3）

| 功能 | 说明 | 命中 |
|---|---|---|
| **F1 主动回忆 + 间隔重复引擎** | SRS 调度器，扩展间隔 (expanding intervals) 自动排程复习；闭卷自测为默认交互 | N1 / LS#1#3#4 / G1 |
| **F2 知识捕获→自测卡片转化** | 把笔记/收藏一键转为提取式卡片，对抗囤积；捕获即进入复习队列 | N1#3 / LS#2 / G3 |
| **F3 AI 苏格拉底式陪练** | 结构化提示：AI 先反问/挑战，用户先产出后校验；拒绝直接代写 | N4 / LS#10 / G2 |
| **F4 双轨进度可视化** | 累计成就 vs 当前目标进度；自我参照（对比过去的你），每图回答"下一步做什么" | N2 / BP§5.1#5.2 |

### P1 — 留存与可持续（对应 N5–N6，G4–G5）

| 功能 | 说明 | 命中 |
|---|---|---|
| **F5 交错出题引擎** | 跨主题混合练习，训练判别性对照（数学/定量尤甚） | LS#9 / LS-B |
| **F6 学期节奏 / 考试周感知调度** | 感知考试临近自动收缩自学、聚焦提分；分散复习反突击 | N5 / G4 / LS#4 |
| **F7 连胜 + 宽容机制** | streak + 冻结/补签，避免意外断签流失；奖励框架随阶段演化 | BP§2.2 / N2 |
| **F8 碎片时间轻量模式** | 核心循环压到 5 分钟内可完成；锚定既有日常触发点（如午餐/睡前） | N6 / BP§3.3 |

### P2 — 增强与合规

| 功能 | 说明 | 命中 |
|---|---|---|
| **F9 焦虑与睡眠模块** | 考前表达性写作（~10 分钟卸载担忧）+ 睡眠/作息提醒 | N5 / LS#5 / LS-C |
| **F10 跨任务优先级仲裁** | 四线统一调度，高优先级抢占、低优先级分解延后 | N6 / P§3.1 |
| **F11 无障碍 + 隐私** | Dynamic Type / 对比度 WCAG AA / 数据最小化、可导出删除 | BP§6 / BP§5.3 |

**MVP 最小闭环建议**：F1 + F2 + F3 + F4。即"捕获知识 → 转为自测 → AI 陪练校验 → 间隔复习 → 看见真实进步"——首次会话内即可完成一次完整学习闭环（命中 BP§1 的 aha moment / TTV）。

## 5. 应遵循的设计原则 (Design Principles)

8 条原则，每条都可作为功能取舍的裁决标准。

| 编号 | 原则 | 含义 | 反面（要避免的） |
|---|---|---|---|
| **DP1** | **合意困难为默认** | 挣扎本身就是学习机制；默认提取/间隔/交错 (LS-A) | 让用户继续重读/高亮等"流畅但无效"操作 |
| **DP2** | **可行动性 > 数据展示** | 每个图表都回答"那我下一步该做什么" (BP§5.1) | 堆砌好看但不促成决策的仪表盘 |
| **DP3** | **内在动机 > 外在奖励** | SDT（自主/胜任/归属）是地基，游戏化只是脚手架 (BP§4) | 把动机建在 streak/XP 上，奖励一停行为就崩 |
| **DP4** | **逻辑优先 + 结构化 + 可视化** | 结论先行→框架→细节→例子→陷阱；推导过程可见；中文默认+英文术语附释义 (P§6) | 冗长散文、寒暄、黑箱推进 |
| **DP5** | **价值优先于花哨 (TTV)** | 引导唯一目标是尽快到达第一次"真正学到东西" (BP§1) | 一次性弹出所有教程、注册前不让体验 |
| **DP6** | **认知负荷感知** | 新手优先 worked examples；削减外在负荷（避免分散注意/冗余）(LS#8) | 对新手直接抛复杂任务、信息过载呈现 |
| **DP7** | **衡量"学会"而非"停留"** | 成功指标是学习成果，不是使用时长；守住伦理红线 (BP§3.4) | 成瘾式设计、虚假的"你真棒"侵蚀信任 |
| **DP8** | **无障碍与隐私从一开始内建** | 不是事后补丁；数据最小化、透明、可控 (BP§6/§5.3) | 上线后再补无障碍；过度采集学习数据 |

## 7. 待验证假设与风险 (Open Assumptions & Risks)

研究综合不能消除以下需在 MVP 阶段验证的假设：

- **A1 单人画像风险**：当前为 n=1 主画像（林深）。需验证目标用户群是否足够大、付费意愿是否成立（P§1 对付费敏感但愿为高 ROI 付费）。
- **A2 合意困难 vs 留存张力**：DP1（让学习"更难"）与 BP§1（降低摩擦）存在张力。需实测：默认强制自测会否在引导期推高流失？建议引导期降难度、核心期上难度。
- **A3 AI 陪练落地难度**：F3 苏格拉底式陪练依赖高质量结构化提示工程，效果随用法分化 (LS#10 证据 ★★★☆☆)，是技术与产品双重风险点。
- **A4 跨学科范围蔓延**：F10 优先级仲裁横跨四领域，易导致 MVP 范围失控。建议先单领域（数学或英语）跑通核心闭环再横向扩展。
- **A5 真实课堂效应量偏小**：多数学习科学效应在真实场景小于实验室 (LS 方法论局限)。预期收益应保守设定。
