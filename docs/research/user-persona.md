## 2. 目标 (Goals)

按优先级排序,对应"AI 开发 > 学业 > 英语 > 金融"的个人价值序。

### 2.1 学业成功 (Academic Success)
- **短期**: GPA 进入专业前 15%,保住保研/留学双通道资格。
- **中期**: 数学 (高数、线代、概率统计) 打到"能推导、能应用"的程度,而非应试背公式。
- **核心诉求**: 把碎片课程知识沉淀为**可检索、可复用的知识体系**,而非考完即忘。

### 2.2 AI 职业能力 (AI Career)
- 成为"金融 + AI"复合人才: 能写生产级 (production-grade) 代码,而非玩具 demo。
- 掌握技术栈: Python → 机器学习 → 量化 (quant) 策略 → LLM 应用 / MCP 工作流。
- **终局画像**: 量化研究员 / AI 产品工程师,能独立从研究到落地 (research → build → deploy)。

### 2.3 英语精通 (English Proficiency)
- IELTS 7.5+,尤其突破写作与口语。
- 能无障碍读英文论文、技术文档、《经济学人》(The Economist)。
- 把英语从"考试科目"转化为"信息获取与表达的工具"。

### 2.4 个人成长 (Personal Growth)
- 建立可持续的自我管理系统 (Personal OS),减少意志力消耗。
- 形成"输入 → 内化 → 输出"的复利学习闭环。
- 维持身心稳定: 抗压、抗倦怠 (burnout)、保持长期动机。

## 3. 痛点 (Pain Points)

### 3.1 时间管理 (Time Management)
- 大一课程密集 + 自学 AI/英语/数学,**时间严重不够分配**。
- 通勤、社团、突发任务导致计划频繁被打断,产生碎片时间但难以利用。
- 缺乏跨任务的优先级仲裁机制,常陷入"什么都想做,什么都没做完"。

### 3.2 考试压力 (Exam Stress)
- 期中/期末集中爆发,与自学计划正面冲突。
- 高数、概率统计推导难度高,临考焦虑导致效率下降。
- 担心 GPA 失守影响保研/留学,形成持续性背景压力。

### 3.3 信息过载 (Information Overload)
- AI 领域更新极快,教程、论文、博客、视频铺天盖地,**不知从何学起**。
- 收藏夹"吃灰",信息囤积 (hoarding) 但未内化。
- 难以分辨信号与噪音 (signal vs. noise),容易被热点带偏主线。

### 3.4 动机波动 (Motivation)
- 自学缺乏即时反馈,容易在"看不到进步"时掉链子。
- 长周期目标 (IELTS、量化) 见效慢,短期诱惑 (娱乐、社交) 强。
- 偶发性自我怀疑: "我是不是不够聪明 / 选错方向了?"

## 4. 日常作息 (Daily Routine)

典型工作日 (学期中):

| 时间 | 活动 | 状态与需求 |
|---|---|---|
| 07:00–08:00 | 起床 + 晨间英语 (50 词 + 阅读理解,邮件推送) | 精力中等,适合轻量输入 |
| 08:00–12:00 | 上课 (专业课 / 数学 / 公共课) | 高认知负荷,需课堂笔记沉淀 |
| 12:00–13:00 | 午餐 + 经济/AI 资讯 (邮件推送 5+5 条) | 碎片时间,适合速读 |
| 13:00–14:00 | 午休 | 恢复 |
| 14:00–17:30 | 上课 / 自习 | 主力学习时段 |
| 18:00–19:00 | 晚餐 + 社团/社交 | 弹性 |
| 19:00–22:30 | **核心自学块**: AI 编程 / 数学推导 / 英语写作 | 高专注,需无干扰环境 + 工具支持 |
| 22:30–23:30 | 复盘 + 次日计划 + 知识归档 | 适合结构化总结 |
| 23:30 | 睡眠 | — |

**洞察**:
- 真正可控的深度学习时间集中在 **19:00–22:30**,这是产品的核心使用窗口。
- 已有自动化习惯 (每日邮件系统 08:00/12:00 推送),说明用户**接受并依赖自动化工作流**。
- 碎片时间 (午餐、通勤) 需要"轻量、可中断"的内容形态。

## 5. 技术栈使用 (Tech Stack Usage)

### 5.1 当前在用
| 类别 | 工具 | 熟练度 |
|---|---|---|
| 操作系统 | Windows 11 | 熟练 |
| AI 助手 | Claude / Claude Code (CLI) | 进阶,作为核心生产力工具 |
| 自动化 | Windows Task Scheduler + Python 脚本 (每日邮件系统) | 中级 |
| 编程语言 | Python (主),JavaScript/TypeScript (学习中) | 初–中级 |
| 知识管理 | Markdown 文件体系 (MEM-CORE / CLAUDE.md 双文件架构) | 进阶 |

### 5.2 目标技术栈 (产品开发方向)
- 前端: Next.js / TypeScript / Tailwind / shadcn/ui
- 数据: PostgreSQL
- 集成: **MCP-first** (Model Context Protocol) 工作流优先
- AI: LLM 应用、Agent、量化策略库

### 5.3 工具哲学 (Tooling Philosophy)
- **Discover before build**: 先找现成 Skill/MCP/CLI,再决定是否自建。
- **复用基础设施**,偏好工作流 (workflow) 而非单点工具。
- 对"高 ROI"工具付费意愿强,对低效冗余工具零容忍。

## 6. 学习偏好 (Learning Preferences)

> 核心: **逻辑优先 (Logic-first) · 结构优先 (Structure-first) · 可视化 (Visual)**

### 6.1 逻辑优先 (Logic-first)
- 拒绝死记硬背,要求"知其所以然": 概念 → 直觉 → 推导 → 例子 → 陷阱 → 应用。
- 数学场景强烈偏好**推导过程可见**,而非直接给公式。

### 6.2 结构化 (Structured)
- 偏好框架化输出: 结论先行 → 框架 → 细节 → 例子 → 陷阱 → 下一步。
- 厌恶冗长散文和寒暄,要求 **diff 优于全量重写**、结构化输出。

### 6.3 可视化 (Visual)
- 偏好表格、流程图、思维导图、对比矩阵承载信息。
- 复杂关系希望用图示而非纯文字描述。

### 6.4 反馈与节奏
- 需要**即时、可验证的反馈** (verifiable success criteria) 来维持动机。
- 接受"失败一次即停、不盲目重试"的工作原则。
- 复杂任务希望先 **outline → 确认 → 执行**,不喜欢黑箱推进。

### 6.5 语言偏好
- 中文为默认,首次出现的英文术语需附中文释义 (EN term + CN gloss)。

## 7. 愿景与抱负 (Aspirations)

### 7.1 三年愿景
- 成为校内"金融 + AI"标杆学生,拥有可展示的项目作品集 (portfolio)。
- 英语达到学术与职业自由使用水平,IELTS 7.5+ 落袋。
- 数学功底足以支撑量化研究的理论需求。

### 7.2 长期抱负
- 进入顶级量化机构 / AI 实验室,或赴海外名校深造。
- 打造属于自己的"个人操作系统 (Personal OS)": 用 AI 与自动化把重复劳动外包,把人力聚焦于高价值的设计、决策与创造。
- 从"学习者"进化为"构建者 (builder)": 不仅消费知识,更能产出生产级成果与影响力。

### 7.3 深层动机 (Underlying Motivation)
- 对**效率与复利**有近乎执念的追求: 相信系统化方法能放大个人能力上限。
- 渴望**确定性与掌控感**: 在快速变化的 AI 时代,通过构建稳固的知识与工具体系来对抗不确定性。

## 8. 设计启示 (Design Implications for AI-Learning-OS)

| 痛点/偏好 | 产品应对策略 |
|---|---|
| 时间稀缺 + 碎片化 | 支持任务优先级仲裁 + 可中断的轻量内容形态 |
| 信息过载 | 信号过滤 + 知识沉淀闭环 (输入→内化→输出),拒绝囤积 |
| 动机波动 | 即时可验证反馈 + 可视化进度,降低意志力消耗 |
| 逻辑/结构优先 | 默认结构化输出 (框架→细节→例子→陷阱),推导过程可见 |
| 可视化偏好 | 表格、流程图、对比矩阵作为信息一等公民 |
| 工具哲学 | MCP-first,复用现有基础设施,工作流优于单点工具 |
| 考试压力 | 学期节奏感知: 考试周自动收缩自学计划、聚焦提分 |
| 语言 | 中文默认 + 英文术语首现附中文释义 |

## 9. 反画像 (Anti-Persona,本产品不优先服务)

- 纯应试型学生: 只要答案不要原理,与"逻辑优先"冲突。
- 被动学习者: 需要外部强推动,缺乏内驱,无法用好自动化工作流。
- 工具排斥者: 拒绝 CLI/自动化,偏好纯 GUI 一键式体验。
