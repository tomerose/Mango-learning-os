## 0. 结论先行 (Thesis)

现有工具各自占据一个象限，**没有一个把"循证学习方法"作为强制默认交互**，也没有一个为"逻辑驱动、跨学科、时间稀缺的金融 AI 学生"做整合。AI Learning OS 的机会 = **PKM（笔记/图谱）+ 间隔重复（闪卡）+ AI 导师 + 计划/复盘** 四件事在一个工作流里闭环，而非四个 App 来回切换。

---

## 1. 竞品矩阵 (Feature Matrix)

| 产品 | 核心定位 | 强项 | 弱项（对我们用户而言） | 可借鉴 |
|---|---|---|---|---|
| **Notion** | 全能工作区 | 灵活数据库、模板生态 | 太通用，学习要自己搭；无间隔重复；移动端重 | 块编辑器、双向链接 |
| **Obsidian** | 本地优先 PKM | 知识图谱、Markdown、插件 | 学习曲线陡；无内建 AI/计划；非学习专用 | 知识图谱、本地优先理念 |
| **Readwise / Reader** | 阅读 + 重温 | 高亮聚合、间隔重温、稍后读 | 偏被动输入；无主动测验/计划 | 间隔重温推送机制 |
| **Anki** | 间隔重复闪卡 | SM-2 算法成熟、极致留存 | UI 老旧；卡片制作枯燥；孤立于学习流 | SM-2 算法（已纳入 schema） |
| **Perplexity** | AI 问答搜索 | 实时检索、引用溯源 | 答完即走，不沉淀；非教学结构 | 结构化引用、追问体验 |
| **Quizlet** | 学生测验 | 题库大、社交化 | 浅层记忆为主；广告多；非深度理解 | 多题型、学习模式切换 |
| **Linear** | 工程项目管理 | 极致流畅、键盘优先、设计标杆 | 非学习领域 | **交互速度、设计语言、命令面板** |
| **Arc / Raycast** | 浏览器 / 启动器 | 沉浸、快捷、美学 | 非学习领域 | 极简美学、快捷操作 |

## 2. 象限定位 (Positioning)

```
        被动输入 ──────────────── 主动产出
          │                          │
  无AI  Readwise  Notion/Obsidian  Anki/Quizlet
          │                          │
  有AI  Perplexity   ★AI Learning OS★
          │     (AI导师 + 计划 + 闪卡 + 图谱 闭环)
```

我们独占的格子：**有 AI × 主动产出 × 学习专用整合**。

---

## 3. 差异化主张 (Differentiators)

1. **循证默认**：主动回忆 + 间隔重复不是"可选功能"，是核心交互（闪卡 SM-2 已建模进 DB）。
2. **跨学科统一**：AI / 经济 / 金融 / 数学 / 英语 同一套进度、XP、图谱体系。
3. **AI 导师有教学结构**：不是问答机器人，遵循"概念→直觉→推导→例子→易错点→应用"管线。
4. **计划-执行-复盘闭环**：日/周/月/学期四级 + 反思系统，对齐长期目标。
5. **Apple 级设计 + Linear 级速度**：学习工具普遍丑、慢；这是我们的体验护城河。

---

## 4. 不做什么 (Anti-Scope)

- ❌ 社交/UGC 题库（Quizlet 路线）—— 稀释专注，引入噪音。
- ❌ 通用工作区（Notion 路线）—— 灵活的代价是"什么都要自己搭"。
- ❌ 插件市场（Obsidian 路线）—— MVP 阶段过度工程。
