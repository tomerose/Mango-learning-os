## 0. 产品定位 (一句话)

> **MangoLearningOS = 你输入目标 → AI接管过程 → 你获得结果的学习操作系统。**

不是"功能集合"，不是"工具平台"，不是"AI聊天机器人"。

## 1. 能力架构 (以能力为单位，不以页面为单位)

### 核心能力层

| 能力 | 定义 | 合并哪些现有功能 |
|------|------|-----------------|
| **C1: 目标理解能力** | 用户说"我想通过XX考试"→系统理解并拆解 | Magic(plan/learn/recommend) + Planner AI生成 |
| **C2: 内容生成能力** | 统一的内容引擎，替代所有独立AI生成 | Magic全部5模式 + exam-package + quiz + flashcard-generate + summary-generate + knowledge-extract |

| **C5: Agent辅导能力** | 统一的学习Agent（替代3个独立对话） | /ai/chat + /ai/agent + /ai/treehole (部分) + MistakeAnalyzer |
| **C6: 规划与执行能力** | 目标→计划→任务→执行的完整链路 | Planner + Tasks + 每日系统 |
| **C7: 成长与留存能力** | XP/Streak/Achievement/每日任务/反思 | Profile + Stats + Streak + Reflection |
| **C8: 项目实践能力** | 学以致用——项目驱动的学习验证 | Projects(Builder/Workspace/Gallery/AI Review) |
| **C9: 心理支持能力** | 学习过程中的情绪管理和动力维护 | Treehole + MindJournal + MoodTracker + CbtReframer |
| **C10: 身份与人格能力** | 个人学习风格识别 + 自适应教学 | VoiceSoul + AgentMemory + 学习偏好 |

## 2. 功能架构 (重构方案)

### 2.1 功能合并方案

| 合并组 | 从 | 到 | 方案 |
|--------|----|----|------|
| AI对话统一 | chat + agent + treehole | **C5: 统一学习Agent** | 一个Agent入口，根据用户意图自动切换对话模式(教学/辅导/陪伴) |
| 内容生成统一 | Magic + exam-package + quiz + flashcard + summary + knowledge-extract | **C2: Content Intelligence Engine** | 一个API + 统一Prompt模板 + 质量评估 + 迭代生成 |
| 考试准备统一 | Magic "考试" + ExamWorkspace + exam-package | **C4: 练习与评估** | 一个入口："我要考试"→自动走完整流程 |
| 计划生成统一 | Magic "计划" + Planner AI | **C6: 规划与执行** | Planner成为唯一入口，Magic去掉计划模式 |
| 推荐统一 | AI Recs(hub) + Magic "推荐" | **C1: 目标理解** | 目标系统自动匹配推荐，不再需要静态推荐卡片 |

### 2.2 页面架构 (从7窗口→4空间)

| 新空间 | 路由 | 能力 | 替代旧窗口 |
|--------|------|------|-----------|
| **今日 (Today)** | `/today` | C6+C7 | Hub + Planner 合并。打开app看到的第一页。每日任务 + 学习进度 + "今天做什么" |
| **学习 (Learn)** | `/learn` | C2+C3+C4+C5 | Agent + Exam 合并。统一的AI导师 + 知识库 + 练习 |
| **实践 (Build)** | `/build` | C8 | Grow项目的Project部分独立。项目驱动的学习验证 |
| **我 (Me)** | `/me` | C7+C9+C10 | Profile + Grow(心理) + DNA。个人成长与心理健康 |

### 2.3 被合并掉的页面

| 旧页面 | 去向 |
|--------|------|
| `/hub` | → `/today` (核心功能) |
| `/agent` | → `/learn` (AI导师) |
| `/exam` | → `/learn` (考试+知识库) |
| `/grow` | → `/build` (项目) + `/me` (心理) |

## 3. Agent 架构 (统一学习Agent)

### 3.1 设计原则

- **一个Agent入口**: 用户不需要选择"和哪个AI对话"
- **上下文感知**: Agent知道用户的学习目标/薄弱领域/最近学习内容
- **模式自适应**: 根据用户意图自动切换（解释概念 / 出题练习 / 错题分析 / 学习规划 / 情绪支持）
- **可执行**: Agent不仅是聊天，可以创建任务、调整计划、生成练习
- **记忆可见**: 用户可以看到Agent记住了什么，可以删除/修改

### 3.2 Agent能力

```
Mango Agent
├── 教学: 6步框架（概念→直觉→推导→例子→易错→下一步）
├── 出题: 根据薄弱领域自动生成练习
├── 规划: 根据目标生成/调整学习计划
├── 分析: 诊断错题根因
├── 总结: 从学习材料生成笔记/闪卡/知识图谱
├── 陪伴: 学习挫折时的情绪支持（"小树"模式）
└── 执行: 创建任务、设置提醒、标记完成
```

### 3.3 Agent Memory 闭环

```
用户对话 → remember() 存储
                ↓
下次对话 ← summarizeContext() 注入 system prompt
                ↓
Agent开口: "上次你卡在特征值分解，今天要练习吗？"
```

## 4. 内容架构 (Content Intelligence Engine)

### 4.1 统一内容生成系统

**原则: 所有模块统一调用。任何内容生成必须包含7个要素。**

| 要素 | 说明 |
|------|------|
| 概念 | 核心概念一句话点明 |
| 案例 | 真实可验证的例子 |
| 步骤 | 关键步骤拆解 |
| 练习 | 1-2个思考题 |
| 答案 | 参考答案+解析 |
| 复习 | 关联的已有知识 |
| 延伸 | 下一步推荐 |

### 4.2 统一Prompt模板系统

```
系统模板
├── 学科Persona (5科 × 1)
├── 输出格式Schema (JSON Schema验证)
├── 用户上下文注入 (AgentMemory + 薄弱领域)
├── 质量自检清单 (AI生成后自我检查)
└── 迭代修正循环 (不满意?→修改→再生成)
```

### 4.3 内容质量保障

| 环节 | 机制 |
|------|------|
| 输入 | 验证用户输入完整性和明确性 |
| 生成 | JSON Schema约束输出结构 |
| 验证 | 解析后验证字段完整性 |
| 反馈 | 用户可评价"有用/无用" |
| 缓存 | 相同topic+相同用户→返回缓存 |
| 降级 | AI失败→返回确定性fallback而非空白 |

## 5. 用户路径 (Goal → Result)

### 5.1 新用户Onboarding

```
1. 欢迎 → "你想通过学习达成什么？"
2. 用户输入目标 → AI拆解为学习路径
3. 用户选择学科 → 系统预设知识体系
4. 进入 /today → 看到第一个每日任务
5. 完成第一个任务 → 解锁streak

```
打开app (/today)
    ↓
看到: "今天的学习任务" (AI根据目标生成)
    ↓
[5秒快感] 完成一个闪卡复习 → XP + 连续天数更新

## 7. 技术实现路径

### Phase 1: 内容引擎统一 (2-3天)

1. 创建 `lib/ai/content-engine.ts` — 统一内容生成API
2. 创建统一的Prompt模板系统 (`lib/ai/templates/`)
3. 添加JSON Schema验证 + 质量自检
4. 迁移Magic + quiz + exam-package到统一引擎
5. 删除旧API路由（合并为`/api/ai/generate`）

### Phase 2: Agent统一 (2-3天)

1. 创建 `lib/ai/agent/` — 统一Agent系统
2. Agent Memory读取闭环
3. 多模式Agent（教学/出题/分析/陪伴）
4. 用户可查看/编辑Agent记忆
5. 合并3个对话API为1个

### Phase 3: 页面重构 (3-4天)

1. 创建 `/today` 页面（合并Hub+Planner核心）
2. `/hub` → 301 redirect to `/today`
3. 创建 `/learn` 页面（统一Agent+Exam+知识库）
4. 创建 `/build` 页面
5. 创建 `/me` 页面（合并Profile+Grow+Dna）

### Phase 4: 留存系统 (2-3天)

1. 每日任务自动生成（基于学习目标和SM-2算法）
2. Streak系统增强（duolingo-style冷冻机制）
3. 学习进度可视化（知识树增长动画）
4. 回顾功能（周/月学习报告）

### Phase 5: 打磨与发布 (1-2天)

1. 旧代码清理、重定向更新
2. 性能优化、bundle size检查
3. 移动端适配
4. 文档更新

## 8. 风险分析

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| 重构期间功能不可用 | 中 | 高 | 分阶段迁移，每阶段完成即部署 |
| 用户不习惯新布局 | 高 | 中 | 保留301重定向，渐进式过渡 |
| AI内容质量不稳定 | 高 | 高 | JSON Schema约束 + fallback + 用户反馈循环 |
| Agent Memory闭环复杂 | 中 | 中 | 先用localStorage实现，验证后再上Supabase |
| 功能合并漏掉边缘case | 中 | 低 | 保留旧代码分支以备回滚 |

## 10. 核心原则 (来自审计+竞品研究的总结)

1. **用户输入目标，系统自动匹配能力** — 不展示功能菜单
2. **打开app 3秒内知道今天要做什么** — /today 是唯一首页
3. **任何AI输出都是结构化+可用+可评价的** — Content Engine统一质量
4. **Agent记得你、了解你、主动帮助你** — Memory闭环
5. **你的知识在增长，你看得见** — 知识资产可视化
6. **5秒快感 + 25分钟深度 = 每日使用循环** — Hook Model设计
7. **学习成果可以展示、分享、导出** — 学习证据
8. **不增加功能，只合并功能。不增加页面，只减少页面。**
