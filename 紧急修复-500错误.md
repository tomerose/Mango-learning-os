# 🔴 紧急修复：Vercel 500 错误

## 问题根因

**环境变量全部为空字符串**:
```
AI_API_KEY=""
AI_BASE_URL=""
AI_MODEL=""
NEXT_PUBLIC_SUPABASE_ANON_KEY=""
NEXT_PUBLIC_SUPABASE_URL=""
```

导致 middleware 错误判断为"已配置"，尝试初始化 Supabase 但传入空值 → 500 错误。

## 立即修复步骤

### 1. 访问 Vercel Dashboard
https://vercel.com/mango-s-projects5/mango-learning-os/settings/environment-variables

### 2. 删除所有空值变量
点击每个变量的 "..." → Delete:
- AI_API_KEY
- AI_BASE_URL
- AI_MODEL
- NEXT_PUBLIC_SUPABASE_ANON_KEY
- NEXT_PUBLIC_SUPABASE_URL
- SUPABASE_SERVICE_ROLE_KEY

### 3. 重新添加正确的值

#### Supabase 配置
1. 访问 https://supabase.com/dashboard/projects
2. 选择你的项目 → Settings → API
3. 复制以下值：

**添加变量**:
- Name: `NEXT_PUBLIC_SUPABASE_URL`
  - Value: `https://[你的项目ID].supabase.co`
  - Environments: ✅ Production

- Name: `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - Value: `eyJ...` (anon public key)
  - Environments: ✅ Production

- Name: `SUPABASE_SERVICE_ROLE_KEY`
  - Value: `eyJ...` (service_role secret)
  - Environments: ✅ Production

#### DeepSeek 配置
1. 访问 https://platform.deepseek.com/api_keys
2. 复制 API Key

**添加变量**:
- Name: `AI_API_KEY`
  - Value: `sk-...` (你的 DeepSeek Key)
  - Environments: ✅ Production

- Name: `AI_BASE_URL`
  - Value: `https://api.deepseek.com`
  - Environments: ✅ Production

- Name: `AI_MODEL`
  - Value: `deepseek-chat`
  - Environments: ✅ Production

## 为什么会出现空值？

Vercel CLI `env pull` 无法拉取加密变量的实际值，只能拉取到空字符串占位符。
这些占位符**不应该**被推送到 Vercel，但可能在某次操作中被误添加。

**正确流程**:
- 手动在 Vercel Dashboard 配置
- 本地通过 Dashboard 查看并手动复制到 `.env.local`
- **永远不要** 通过 CLI 推送环境变量
