# 配置 Vercel 环境变量指南

## 第一步：获取 Supabase 凭证

### 1. 访问 Supabase Dashboard
https://supabase.com/dashboard/projects

### 2. 选择你的项目（或创建新项目）
- 如果没有项目，点击 "New Project"
- 输入项目名称（如 `mango-learning-os`）
- 选择区域（推荐 Singapore 或 Tokyo）
- 创建后等待 2-3 分钟初始化

### 3. 获取 API 凭证
进入项目 → 左侧菜单 **Settings** → **API**

复制以下值：
- **Project URL**: `https://xxxxx.supabase.co`
- **anon public**: `eyJhbGci...` (长字符串)
- **service_role secret**: `eyJhbGci...` (更长的字符串，需要点击 "Reveal" 显示)

### 4. 初始化数据库
进入 **SQL Editor** → 新建查询 → 粘贴以下内容：

```sql
-- 从项目文件读取完整 schema
-- 路径: D:\Claudecoda学习\AI-Learning-OS\docs\architecture\database-schema.sql
```

或访问本地文件：
`D:\Claudecoda学习\AI-Learning-OS\docs\architecture\database-schema.sql`

复制全部内容到 SQL Editor 并执行（Run）。

## 第三步：配置 Vercel 环境变量

### 1. 访问 Vercel 项目设置
https://vercel.com/mango-s-projects5/mango-learning-os/settings/environment-variables

**变量 1**:
- Key: `NEXT_PUBLIC_SUPABASE_URL`
- Value: `https://xxxxx.supabase.co` （从第一步获取）

- Value: `eyJhbGci...` （anon public key）
- Environments: ✅ Production, ✅ Preview, ✅ Development

**变量 3**:

- Value: `eyJhbGci...` （service_role secret）
- Environments: ✅ Production

### 3. 添加 DeepSeek 变量

## 常见问题

### Q: Supabase 初始化失败
**A**: 检查 SQL 是否完整执行。重新运行 `database-schema.sql`。

### Q: DeepSeek API 调用失败
**A**: 
1. 检查 API Key 是否正确复制（无空格）
2. 访问 DeepSeek Dashboard 确认 Key 有效且有额度
3. 查看 Vercel Logs 确认错误信息

### Q: 登录后数据丢失
**A**: 首次登录会自动 seed 演示数据，后续操作才是真实数据。

### Q: 考试模式仍显示"还没有测验记录"
**A**: 需要先在 **AI 导师 → 测验** 完成 1-2 次测验生成数据。
